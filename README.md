## ⟡ Featured Projects

<details open>
<summary><b>◆ Production RAG Eval</b> — open-source RAG pipeline + evaluation harness</summary>

<br/>

Production retrieval-augmented generation over a 400-document corpus with a 30-query hand-labeled eval suite — **every reported number is backed by a committed JSON receipt** in the repo. Built native Python first, then benchmarked LangChain and a LangGraph multi-agent variant against it to quantify the real cost of abstraction.

| Attribute | Detail |
|-----------|--------|
| **Stack** | Python 3.12 · Mistral API · pgvector / Postgres 16 (HNSW) · FastAPI · LangGraph · LangChain · pytest |
| **Scale** | 400-doc corpus · 30-query labeled eval set · 60 RAG calls + 54 judge calls per run |
| **Performance** | precision@5 `0.68` · recall@5 `0.73` · MRR `0.92` · p50 latency `~1.2s` (small model) |
| **Security** | refusal eval `3/3` on negative tests · `0` hallucinations · citation accuracy `5.0` · secrets via `.env`, no keys committed |
| **Impact** | Data-driven model selection: `mistral-small` matched `mistral-large` (4.56 vs 4.59) at ~10× lower cost; proved multi-agent added 1.68× cost / 3.5× latency with **no** quality gain |
| **Repository** | [github.com/openatlaspro-AI/production-rag-eval](https://github.com/openatlaspro-AI/production-rag-eval) · [live demo](https://openatlaspro-rag-eval.streamlit.app) |

This project demonstrates the discipline most RAG demos skip: a real evaluation harness. It measures quality, latency, and cost at every stage, ships an honest negative result on multi-agent decomposition, and keeps the single-agent path in production because the data said so.

</details>

<details>
<summary><b>◆ Quant Research Platform</b> — systematic trading research with anti-self-deception validation</summary>

<br/>

End-to-end research platform for testing systematic trading signals on US equities — built as a portfolio engineering project, explicitly **not** a money-making system. Its defining property: it tells you when something *doesn't* work. The validation pipeline is designed to catch the author fooling themselves.

| Attribute | Detail |
|-----------|--------|
| **Stack** | Python · DuckDB · pandas / numpy · point-in-time data layer · interactive dashboard |
| **Scale** | 165-ticker universe (large/mid/small caps + sector & macro ETFs) · multi-horizon features |
| **Rigor** | one-bar execution lag · stochastic look-ahead tests on every feature · walk-forward validation · realistic costs (half-spread + slippage + SEC fee + borrow) + 2× cost stress |
| **Honesty** | Deflated Sharpe (Bailey & López de Prado) for multiple-testing correction; documented arc where only 1 of 3 signal families had positive out-of-sample Sharpe — and that one roughly breakeven net of costs |
| **Repository** | [github.com/openatlaspro-AI/quant-research-platform](https://github.com/openatlaspro-AI/quant-research-platform) |

The same engineering value that runs through my AI work: measure honestly, prevent look-ahead and leakage, and report the result even when it's negative.

</details>
